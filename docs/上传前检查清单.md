# 上传前检查清单

这份清单是给你真正 `git init` / `git add` / `git push` 之前最后过一遍用的。

## 必查项

1. 确认仓库根目录就是这个上传版目录，而不是原始工作区根目录。
2. 确认没有把 `.cache/`、`.tmp*/`、`年报/`、`2001-2025/` 这些大目录误拷进来。
3. 确认没有真实 API Key，只保留 `.env.example`。
4. 确认没有 `progress_status.json`、`extract_log.csv`、`task_queue.sqlite` 之类运行期产物。
5. 确认没有 PDF、ZIP、日志、人工修复结果、批量 JSON 汇总结果。
6. 确认 `README.md` 里的运行方式仍然对应你现在的脚本和目录设计。
7. 确认 `reference_data/` 中的文件都是你愿意公开的小体积参考文件。

## 建议项

1. 如果你打算真正开源，而不是只做展示，请把 `LICENSE.md` 替换成标准开源协议。
2. 如果你担心别人误解“仓库内不含原始数据”，可以在 GitHub 仓库描述里明确写上：
   `code-only public mirror; raw annual reports and cached outputs are excluded`.
3. 如果你希望首页更像研究项目而不是脚本仓库，可以补一张流程图和一张结果示意图。
