# 数据目录约定

这个公开仓库只放代码和说明，不放大体量运行数据。实际跑任务时，建议单独准备一个本地数据工作区。

## 推荐结构

```text
D:\fire-data\
  年报\
    下载年报_fulltext\
      2001\
      2002\
      ...
      2025\
  .cache\
    qwen_pdf_markdown_remaining\
      output_markdown\
  2001\
  2002\
  ...
  2025\
  CPI指数汇总.csv
  hs300_with_ipo_status.csv
  hs300_history.json
```

## 两类目录的职责

### 1. 代码仓库

只放这些：

- `scripts/`
- `docs/`
- `configs/`
- `reference_data/`
- `README.md`
- `requirements.txt`

### 2. 数据工作区

只放这些：

- 原始 PDF 年报
- Marker / OCR / Markdown 缓存
- 批量抽取输出
- 按年份生成的 CSV
- 回测结果图表
- 监控状态文件

## 关键参数怎么传

如果代码仓库和数据工作区已经分离，运行脚本时要显式传路径。

常见参数包括：

- `--base-dir`
- `--pdf-root`
- `--markdown-root`
- `--year-csv-root`
- `--out-dir`
- `--cpi-csv`
- `--status-csv`
- `--history-json`

## 一个典型例子

```powershell
python scripts\repair_gemma_markdown_financials.py `
  --base-dir D:\fire-data `
  --markdown-root D:\fire-data\.cache\qwen_pdf_markdown_remaining\output_markdown `
  --out-dir D:\fire-data\.tmp_gemma_markdown_financials_full `
  --year-csv-root D:\fire-data `
  --api-base-url http://127.0.0.1:1234/v1 `
  --api-key-env LM_STUDIO_API_KEY
```

## reference_data 目录的定位

`reference_data/` 里的文件只是轻量参考数据，用来说明格式和减少重新摸索成本。它们不是完整生产环境，也不是全部研究结果。

如果某个脚本默认从项目根目录读取 `CPI指数汇总.csv` 或 `hs300_with_ipo_status.csv`，你有两种做法：

1. 把对应文件复制到你的数据工作区根目录。
2. 运行时通过参数显式指定路径。

推荐第二种，目录职责更清晰。
